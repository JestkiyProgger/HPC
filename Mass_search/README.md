# Лабораторная работа 4

Задача: Реализовать алгоритм массового поиска подстрок произвольной длины в буфере данных с использованием CPU и GPU, сравнить производительность.

Язык: Python

Среда выполнения: Google Colab

GPU: NVIDIA Tesla T4

Библиотека: PyCUDA - для взаимодействия с CUDA на низком уровне.

## Проделанная работа

### 1. Реализация на CPU

- Создание рабочей матрицы R размером n × h, инициализация длинами подстрок;
- Построение хеш-таблицы символов: для каждого символа (0-255) список пар (индекс подстроки, позиция в подстроке);
- Проход по буферу с декрементом соответствующих элементов матрицы;
- Поддержка двух режимов поиска:
  - Поиск всех позиций подстрок;
  - Поиск только факта присутствия.

### 2. Реализация на GPU

- CUDA ядро: 
  - Каждый поток обрабатывает один символ буфера (размер блока 256 потоков);
  - Атомарные операции atomicAdd для безопасного обновления матрицы;
  - Разделение на два режима работы через параметр find_all_positions.

- Оптимизации:
  - Использование атомарных операций для избежания гонок данных;
  - Блочная обработка для эффективного использования кэша;
  - Передача всех структур в constant memory через параметры ядра.

### 3. Измерение производительности

- Время на CPU измеряется с помощью time.perf_counter();
- Время на GPU измеряется с помощью time.time() с синхронизацией cuda.Context.synchronize().

### 4. Сохранение входных данных

Реализована функция save_input_data для сохранения:
- buffer.bin - бинарный файл с буфером поиска;
- patterns.bin - бинарный файл с подстроками (количество подстрок, затем для каждой: длина и данные);
- info.txt - текстовый файл с метаинформацией.
